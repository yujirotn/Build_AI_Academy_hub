# ツール解説系台本の書き方

## 概要

ツール解説系台本は、特定のAIツールの機能・料金・使い方を網羅的に紹介する台本です。

**例**: Difyの使い方、NotebookLM完全解説、Canva AI機能紹介

**想定時間**: 15〜20分

---

## 台本フォーマット（必須）

すべての台本は以下の形式で作成すること。

### 基本構造

```markdown
# [ツール名]の使い方

ジャンル: ツール解説系
想定時間: XX〜XX分
作成日: YYYY年M月D日

## 1. オープニング

【スタート】こんにちは、Build AI Academyのたんのです。

今回は、[開発元]が提供する「[ツール名]」をご紹介します。
（画面：[ツール名]のロゴ）

[本文...]

## 2. ツール概要

[本文...]

### [サブセクション名]

[本文...]

## 3. プラン・料金体系

[本文...]

## 4. 制約・制限事項

[本文...]

## 5. 実演：[機能名]

（画面：[ツール]のトップページ）

[本文...]

## 6. ユースケース

[本文...]

## 7. 注意点

[本文...]

## 8. クロージング

[まとめの本文...]

【終わり】それでは次の講座で会いましょう。ありがとうございました。
```

### フォーマットルール

| 項目 | ルール |
|------|--------|
| タイトル | `# [ツール名]の使い方`（H1） |
| 冒頭情報 | ジャンル・想定時間・作成日のみ。箇条書きにしない |
| セクション | `## N. セクション名`（1から連番） |
| サブセクション | `### サブセクション名` |
| 区切り線 | **使用しない**（`---` は入れない） |
| 【スタート】 | 1. オープニングの冒頭、最初の発話の前に配置 |
| 【終わり】 | 最終セクションの最終文の冒頭に配置 |
| 画面指示 | `（画面：〜）` 形式。セクション冒頭や重要な場面で使用 |

---

## 台本構成（8セクション）

### 1. オープニング

- **開始文言**: 「【スタート】こんにちは、Build AI Academyのたんのです」
- **ツール名の提示**: 今回紹介するツールを明示
- **難易度の提示**: 「このツールは〇〇向けです」と最初に伝える
  - 初心者向け / 中級者向け / 上級者向け
  - 判断理由も一言添える（例：「プログラミング知識は不要なので」「設定項目が多いので」）
- **住宅業界での活用イメージ**: ツールの機能を住宅業界の現場に寄り添って説明
  - NG: 「AIチャットボットが作れるツールです」（機能の説明だけ）
  - OK: 「例えば、お客様からよくある質問に自動で答えるチャットボットを作ったり、社内の建材情報を検索できるAIアシスタントを作ったりできます」

### 2. ツール概要

- **ツールの基本情報**
  - 正式名称
  - 開発元
  - サービス開始時期（わかれば）
- **一言で言うと何ができるツールか**
  - 専門用語を使う場合は必ず説明を添える
- **主要機能の紹介**
  - 3〜5つ程度の主な機能を紹介
  - 各機能は住宅業界での使い道と一緒に説明する

### 3. プラン・料金体系

- **無料プランの説明**
  - 何ができるか、何ができないか
  - 「まずは無料で試せます」など、導入ハードルの低さを伝える
- **有料プランの説明**
  - プラン名と価格（月額/年額）
  - 各プランの主な機能差
  - 「どんな人にどのプランがおすすめか」を添える
- **トライアル情報**（あれば）
- **注意事項**
  - 料金は調査時点のものである旨を明記
  - 価格改定の可能性に触れる

```
※料金は2024年〇月時点の情報です。最新の料金は公式サイトでご確認ください。
```

### 4. 制約・制限事項

- **無料プランの制限**
  - 利用回数、機能制限など
- **全体的な制約**
  - データ容量の上限
  - API制限やレート制限（該当する場合）
- **対応環境**
  - 対応ブラウザ
  - モバイルアプリの有無
  - 日本語対応状況
- **その他の制約**
  - データ保存期間
  - エクスポート制限など

※専門用語（API、レート制限など）は必ず説明を添える

### 5. 画面操作の実演

このセクションが台本のメインパートです。

#### 5-1. アカウント作成・初期設定

```
【アカウント作成の手順】
1. 公式サイトにアクセス
2. 「サインアップ」をクリック
3. メールアドレスを入力
4. パスワードを設定
5. 確認メールのリンクをクリック
6. 初期設定を完了
```

#### 5-2. 基本操作（メイン機能）

ツールの主要機能について、それぞれ操作手順を記載する。

```
【〇〇機能の使い方】
1. ダッシュボードから「〇〇」をクリック
2. 「新規作成」を選択
3. 必要な情報を入力
4. 設定を確認
5. 「保存」または「実行」をクリック
```

※機能ごとにセクションを分ける
※無料版と有料版で操作が異なる場合は両方記載する

#### 5-3. よく使う設定・カスタマイズ

```
【おすすめの初期設定】
1. 言語設定を日本語に変更
2. 通知設定の調整
3. その他便利な設定
```

### 6. ユースケース

- **一般的な活用例を2〜3個紹介**
  - 具体的なシーンとともに説明
- **住宅業界での活用可能性**
  - 「住宅会社さんだと、こんな使い方もできそうですね」程度で軽めに
  - 1〜2個の提案に留める
  - ※詳しいユースケースは別動画で扱う旨を伝えてもOK

### 7. 注意点

- **導入時の注意**
  - 始める前に知っておくべきこと
- **運用上の注意**
  - 使い続ける上で気をつけること
- **よくあるつまずきポイント**
  - 初心者がハマりやすい点
  - その回避方法も添える

### 8. クロージング

- **ツールの価値を再確認**
  - 住宅業界での活用メリットを改めて伝える
- **次のステップの提案**
  - 「まずは無料プランで触ってみてください」
  - 「具体的な活用方法は別の講座で紹介します」など
- **終了文言**: 「【終わり】それでは次の講座で会いましょう。ありがとうございました。」

## 台本作成時の注意事項

### 情報の正確性

- 料金・制限事項は必ず公式サイトで確認する
- 調査日を明記する
- 頻繁に変わる情報は「最新情報は公式サイトでご確認ください」と添える

### 表現のルール

- **専門用語・横文字には必ず説明を添える**
  - NG: 「APIを使って連携できます」
  - OK: 「API、つまり他のサービスとデータをやり取りする仕組みを使って連携できます」
- **住宅業界の現場に寄り添った説明を心がける**
  - 抽象的な機能説明ではなく、現場での使用シーンを描写する
- **初心者にも理解しやすい表現を使う**
  - IT用語を使いすぎない
  - 操作手順は具体的に

### 構成のバランス

- 画面操作の実演パートに最も時間を割く（全体の約40%）
- 料金・制限の説明は正確に、でも長くなりすぎないように
- ユースケースは軽めに（詳細は別動画へ誘導）

## 過去の台本作成での学び

### Google AI Studio台本作成時（2026-01-05）

#### 1. レート制限の具体的な数値は記載しない

公式ドキュメントには「1日○回」などの具体的な制限値が明記されていないことが多い。また、制限値は頻繁に変更される。

**NG**: 「Gemini 3 Proは1日25回、Gemini 3 Flashは1日500回」
**OK**: 「Pro系のモデルは制限が厳しめ、Flash系は比較的多く使えます。制限値はモデルや時期によって変動するため、実際に使いながら確認してください。」

#### 2. 専門用語は徹底的に避ける

「API」「レート制限」などの用語は、視聴者には伝わらない可能性が高い。必要な場合は説明を添えるか、別の表現に置き換える。

**NG**: 「Gemini APIと一緒に公開されました」
**OK**: 「2023年末頃に公開されました」

#### 3. 構成は「概要 → 実演 → 注意事項」がベター

実演に入る前の説明（概要・料金・制限など）が長くなると、視聴者が離脱しやすい。簡潔な概要のあと、すぐに実演に入り、料金や注意点は実演後に説明する構成が効果的。

#### 4. 実際の画面UIに基づいて機能を説明する

公式ドキュメントだけでなく、実際のツール画面を確認して機能構成を把握する。画面に表示されているタブやメニュー名に沿って説明すると、視聴者が迷わない。

例：Google AI Studioのチャット画面には「Gemini」「Live」「Images」「Video」「Audio」の5つのタブがある → それぞれの機能を解説
