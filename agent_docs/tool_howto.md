# ツール解説系台本の書き方

## 概要

ツール解説系台本は、特定のAIツールの機能・料金・使い方を網羅的に紹介する台本です。

**例**: Difyの使い方、NotebookLM完全解説、Canva AI機能紹介

**想定時間**: 15〜20分

---

## 台本の出力形式

### 基本ルール

- **シンプルな文章形式**で記載する（表形式は使わない）
- **時間の記載は不要**
- **画面指示**は必要な場合のみ、該当箇所の直後にカッコ書きで簡潔に記載する
  - 例: `（画面：Mixboardのトップページ）`
  - 例: `（画面：画像生成の様子）`
- セクション見出しは `## 1. オープニング` のようにMarkdownの見出しで記載する

### 出力形式の例

```markdown
## 1. オープニング

【スタート】こんにちは、Build AI Academyの〇〇です。

今回は、Googleが提供するAIツール「Mixboard」をご紹介します。
（画面：Mixboardのロゴ）

このツールは初心者向けです。プログラミングの知識は一切不要で、日本語で指示を入力するだけで使えます。

## 2. ツール概要

では、Mixboardの基本情報から見ていきましょう。
```

### NG例（使用禁止）

```markdown
| 時間 | ナレーション | 画面指示 |
|------|-------------|----------|
| 0:00 | 【スタート】こんにちは... | タイトル画面 |
```

## 台本構成（8セクション）

### 1. オープニング

- **開始文言**: 「【スタート】こんにちは、Build AI Academyの〇〇です」
- **ツール名の提示**: 今回紹介するツールを明示
- **難易度の提示**: 「このツールは〇〇向けです」と最初に伝える
  - 初心者向け / 中級者向け / 上級者向け
  - 判断理由も一言添える（例：「プログラミング知識は不要なので」「設定項目が多いので」）
- **住宅業界での活用イメージ**: ツールの機能を住宅業界の現場に寄り添って説明
  - NG: 「AIチャットボットが作れるツールです」（機能の説明だけ）
  - OK: 「例えば、お客様からよくある質問に自動で答えるチャットボットを作ったり、社内の建材情報を検索できるAIアシスタントを作ったりできます」

### 2. ツール概要

- **ツールの基本情報**
  - 正式名称
  - 開発元
  - サービス開始時期（わかれば）
- **一言で言うと何ができるツールか**
  - 専門用語を使う場合は必ず説明を添える
- **主要機能の紹介**
  - 3〜5つ程度の主な機能を紹介
  - 各機能は住宅業界での使い道と一緒に説明する

### 3. プラン・料金体系

- **無料プランの説明**
  - 何ができるか、何ができないか
  - 「まずは無料で試せます」など、導入ハードルの低さを伝える
- **有料プランの説明**
  - プラン名と価格（月額/年額）
  - 各プランの主な機能差
  - 「どんな人にどのプランがおすすめか」を添える
- **トライアル情報**（あれば）
- **注意事項**
  - 料金は調査時点のものである旨を明記
  - 価格改定の可能性に触れる

```
※料金は2024年〇月時点の情報です。最新の料金は公式サイトでご確認ください。
```

### 4. 制約・制限事項

- **無料プランの制限**
  - 利用回数、機能制限など
- **全体的な制約**
  - データ容量の上限
  - API制限やレート制限（該当する場合）
- **対応環境**
  - 対応ブラウザ
  - モバイルアプリの有無
  - 日本語対応状況
- **その他の制約**
  - データ保存期間
  - エクスポート制限など

※専門用語（API、レート制限など）は必ず説明を添える

### 5. 画面操作の実演

このセクションが台本のメインパートです。

#### 5-1. アカウント作成・初期設定

```
【アカウント作成の手順】
1. 公式サイトにアクセス
2. 「サインアップ」をクリック
3. メールアドレスを入力
4. パスワードを設定
5. 確認メールのリンクをクリック
6. 初期設定を完了
```

#### 5-2. 基本操作（メイン機能）

ツールの主要機能について、それぞれ操作手順を記載する。

```
【〇〇機能の使い方】
1. ダッシュボードから「〇〇」をクリック
2. 「新規作成」を選択
3. 必要な情報を入力
4. 設定を確認
5. 「保存」または「実行」をクリック
```

※機能ごとにセクションを分ける
※無料版と有料版で操作が異なる場合は両方記載する

#### 5-3. よく使う設定・カスタマイズ

```
【おすすめの初期設定】
1. 言語設定を日本語に変更
2. 通知設定の調整
3. その他便利な設定
```

### 6. ユースケース

- **一般的な活用例を2〜3個紹介**
  - 具体的なシーンとともに説明
- **住宅業界での活用可能性**
  - 「住宅会社さんだと、こんな使い方もできそうですね」程度で軽めに
  - 1〜2個の提案に留める
  - ※詳しいユースケースは別動画で扱う旨を伝えてもOK

### 7. 注意点

- **導入時の注意**
  - 始める前に知っておくべきこと
- **運用上の注意**
  - 使い続ける上で気をつけること
- **よくあるつまずきポイント**
  - 初心者がハマりやすい点
  - その回避方法も添える

### 8. クロージング

- **ツールの価値を再確認**
  - 住宅業界での活用メリットを改めて伝える
- **次のステップの提案**
  - 「まずは無料プランで触ってみてください」
  - 「具体的な活用方法は別の講座で紹介します」など
- **終了文言**: 「【終わり】それでは次の講座で会いましょう。ありがとうございました。」

## 台本作成時の注意事項

### 情報の正確性

- 料金・制限事項は必ず公式サイトで確認する
- 調査日を明記する
- 頻繁に変わる情報は「最新情報は公式サイトでご確認ください」と添える

### 表現のルール

- **専門用語・横文字には必ず説明を添える**
  - NG: 「APIを使って連携できます」
  - OK: 「API、つまり他のサービスとデータをやり取りする仕組みを使って連携できます」
- **住宅業界の現場に寄り添った説明を心がける**
  - 抽象的な機能説明ではなく、現場での使用シーンを描写する
- **初心者にも理解しやすい表現を使う**
  - IT用語を使いすぎない
  - 操作手順は具体的に

### 構成のバランス

- 画面操作の実演パートに最も時間を割く（全体の約40%）
- 料金・制限の説明は正確に、でも長くなりすぎないように
- ユースケースは軽めに（詳細は別動画へ誘導）

## 品質チェックリスト

### 共通チェック項目

- [ ] 開始文言が「【スタート】こんにちは、Build AI Academyの〇〇です」になっている
- [ ] 終了文言が「【終わり】それでは次の講座で会いましょう。ありがとうございました。」になっている
- [ ] 専門用語・横文字にすべて説明が付いている
- [ ] 全体が初心者にも理解しやすい表現になっている
- [ ] 住宅会社目線の内容が含まれている

### ツール解説系専用チェック項目

- [ ] オープニングで難易度が提示されている
- [ ] オープニングで住宅業界での活用イメージが伝えられている
- [ ] ツール概要で主要機能が3〜5つ紹介されている
- [ ] 料金体系が無料/有料ともに説明されている
- [ ] 料金の調査時点が明記されている
- [ ] 制約・制限事項が漏れなく記載されている
- [ ] 画面操作の手順がステップ形式で書かれている
- [ ] 無料版/有料版の違いが明確になっている
- [ ] ユースケースが軽めに（2〜3個）紹介されている
- [ ] 注意点・つまずきポイントが記載されている

台本作成時には、以下の情報収集レポートを参照してください：

- ツール概要
- プラン・料金体系
- 制約・制限事項
- 難易度評価
- 画面・操作関連情報
- ユースケース
- 注意点
- 情報ソース一覧

※情報収集レポートは別途「AIツール情報収集エージェント」が作成したものを使用
