## 概要

このガイドは、E-learning講座や研修の台本から、スライド資料の詳細アウトラインを作成するための指示書です。

**重要**: スライドアウトラインは**詳細に**作成する。スライド制作者が迷わず作業できるレベルの具体性を目指す。

---

## 入力と出力

### 入力
- 講座・研修の台本（テキスト形式）

### 出力
- スライドごとの詳細設計書（以下を含む）
  - スライドタイトル
  - レイアウト指定
  - 掲載テキスト（見出し、本文、強調箇所）
  - イラスト・図解の具体的な内容と配置
  - 補足・注記

## 台本タイプ別スライド枚数目安

| 台本タイプ | 想定時間 | スライド枚数目安 |
|-----------|---------|----------------|
| ツール解説系 | 15〜20分 | **25〜35枚** |
| ユースケース系 | 10〜15分 | **15〜25枚** |
| 概念系 | 10〜15分 | **15〜20枚** |
| 比較系 | 15〜25分 | **25〜30枚** |

**基本計算**: 1分あたり1.5〜2枚を目安とする

## 詳細化の基本方針

### 1. 導入部分は必ず詳細化する

導入部分（オープニング〜ツール紹介）は以下のように分割する：

| セクション | 分割後のスライド |
|-----------|----------------|
| 課題提起 | ① こんな経験ありませんか？（課題列挙）<br>② その悩み、〇〇で解決できます（解決策提示） |
| この動画の内容 | ③ この動画でわかること（チェックリスト形式） |
| ツール紹介 | ④ 〇〇とは？（定義）<br>⑤ 住宅業界での活用イメージ（Before/After）<br>⑥ ツール基本情報（カード形式） |

### 2. 機能紹介は個別スライドに分割する

複数の機能がある場合：

- **概要スライド**: 機能一覧を円形/横並びで表示（1枚）
- **個別機能スライド**: 各機能を1枚ずつ詳細に説明（機能数分）

```
例: 5つの機能がある場合
- スライドA: 主な機能（5つ）- 概要
- スライドB: 機能① 画像生成
- スライドC: 機能② 画像編集
- スライドD: 機能③ 画像合成
- スライドE: 機能④ バリエーション
- スライドF: 機能⑤ プレゼン作成
→ 合計6枚
```

### 3. 活用例はシーン別に分割する

活用例は1枚にまとめず、シーン別に分割する：

```
NG: 活用例（3シーンを1枚に詰め込み）

OK:
- スライドA: 活用例① インテリア提案
- スライドB: 活用例② 打ち合わせでのリアルタイム調整
- スライドC: 活用例③ 展示場・モデルハウス企画
→ 合計3枚
```

### 4. 実演部分は簡素でOK

【実演】スライドは、実際の画面操作で補完されるため、簡潔に記載してよい：

- 手順は表形式で簡潔に
- 画面イメージの詳細説明は不要
- 「※実際の画面で実演します」の注記を入れる

```
例: 実演スライドの簡素化
- 【実演】アクセス〜ログイン: 4ステップを1枚で
- 【実演】画像編集・合成・バリエーション: 3機能を1枚で
```

### 5. 職種別おすすめを含める

住宅業界向けコンテンツでは、職種別おすすめスライドを追加する：

| 職種 | 対象者 |
|-----|-------|
| 👔 営業担当 | お客様対応、提案資料 |
| 📐 設計担当 | イメージ検討、方向性確認 |
| 💼 事務担当 | 帳票作成、データ整理 |
| 🏗️ 現場監督 | 仕様書確認、安全管理 |
| 🏢 企画担当 | コンセプト検討、社内プレゼン |

## スライド作成の基本原則

### 1. 情報の分割
- **1スライド1メッセージ**を原則とする
- 台本の1セクションが複数のメッセージを含む場合は、複数スライドに分割する
- 聴衆が5秒で要点を把握できる情報量に抑える

### 2. テキストの簡潔化
- 台本の話し言葉を、箇条書きやキーワードに変換する
- 1スライドのテキスト量は最大5〜7行を目安とする
- 重要なキーワードは**太字**や色で強調する

### 3. ビジュアル優先
- すべてのスライドに、イラスト・図解・アイコンのいずれかを含める
- テキストだけのスライドは原則として作らない
- 「百聞は一見にしかず」を意識する

## イラスト・図解の設計ガイドライン

### 1. 必ずビジュアルを入れるシーン

| シーン | 推奨ビジュアル |
|-------|---------------|
| 課題・問題提起 | 困っている人物イラスト、NGマーク、吹き出し |
| 解決策の提示 | ツールのロゴ/画面、電球アイコン、フロー図 |
| メリット説明 | チェックマーク、上昇グラフ、時計（時短） |
| 比較（Before/After） | 左右対比イラスト、矢印、表情の変化 |
| 機能紹介 | アイコン付きカード、放射図、サンプル画像 |
| 手順説明 | 番号付きステップ図、画面キャプチャ |
| 活用例 | シーンを表すイラスト、会話吹き出し |
| 注意事項 | 警告マーク、情報アイコン |
| まとめ | アイコン付きリスト、チェックリスト |

### 2. 図解の種類と使い分け

| 図解タイプ | 用途 | 例 |
|-----------|------|-----|
| フローチャート | プロセス、手順の説明 | 入力→処理→出力 |
| 比較表 | Before/After、選択肢の比較 | 従来 vs AI活用 |
| 放射図 | 中心概念と関連要素 | ツールを中心に機能を配置 |
| タイムライン | 時系列の変化 | 導入前→導入後の変化 |
| ピラミッド/階層図 | 優先度、構造の説明 | 重要度の高い順に配置 |
| マトリクス | 2軸での分類・比較 | 難易度×効果 |
| カード形式 | 複数項目の並列表示 | 機能一覧、ポイント一覧 |

### 3. アイコン・イラストの調達先（参考）

| 種類 | 調達先例 | 用途 |
|-----|---------|------|
| アイコン | Lucide、Flaticon、Icons8、PowerPoint内蔵 | 機能説明、注意点、ポイント |
| 人物イラスト | いらすとや、ソコスト、Storyset、unDraw | 課題提起、活用シーン |
| 画面キャプチャ | 実際のツール画面をスクリーンショット | 実演スライド |
| フロー図・図解 | PowerPoint/Figmaの図形で作成 | 機能説明、プロセス |
| ロゴ | 各ツール公式サイト | タイトル、基本情報 |
| サンプル画像 | ツールで生成した画像 | 機能紹介、活用例 |

## 作成手順

### Step 1: 台本の構造分析
1. 台本を読み、セクション（章立て）を把握する
2. 各セクションの**目的**（課題提起/解決策/メリット/手順など）を特定する
3. キーメッセージを抽出する

### Step 2: スライド分割
1. **導入部分は詳細化**する（課題提起、解決策、この動画でわかること）
2. **機能紹介は個別スライド**に分割する
3. **活用例はシーン別**に分割する
4. **実演部分は簡素化**してよい
5. タイトル・まとめ・エンディングを追加する
6. 全体のスライド枚数を確認（台本タイプ別目安を参照）

### Step 3: 各スライドの詳細設計
1. スライドタイプを選択（上記テンプレートから）
2. テキスト内容を簡潔化して記入
3. イラスト・図解の内容を**具体的に**記述
4. レイアウトを指定

### Step 4: 全体レビュー
1. 流れに一貫性があるか確認
2. ビジュアルの抜け漏れがないか確認
3. 情報量のバランスを確認（詰め込みすぎていないか）
4. 職種別おすすめが含まれているか確認
