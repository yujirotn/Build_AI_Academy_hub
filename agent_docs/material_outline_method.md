# スライド詳細アウトライン作成ガイド

## 概要

このガイドは、E-learning講座や研修の台本から、スライド資料の詳細アウトラインを作成するための指示書です。

---

## 入力と出力

### 入力
- 講座・研修の台本（テキスト形式）

### 出力
- スライドごとの詳細設計書（以下を含む）
  - スライドタイトル
  - レイアウト指定
  - 掲載テキスト（見出し、本文、強調箇所）
  - イラスト・図解の具体的な内容と配置
  - 補足・注記

---

## スライド作成の基本原則

### 1. 情報の分割
- **1スライド1メッセージ**を原則とする
- 台本の1セクションが複数のメッセージを含む場合は、複数スライドに分割する
- 聴衆が5秒で要点を把握できる情報量に抑える

### 2. テキストの簡潔化
- 台本の話し言葉を、箇条書きやキーワードに変換する
- 1スライドのテキスト量は最大5〜7行を目安とする
- 重要なキーワードは**太字**や色で強調する

### 3. ビジュアル優先
- すべてのスライドに、イラスト・図解・アイコンのいずれかを含める
- テキストだけのスライドは原則として作らない
- 「百聞は一見にしかず」を意識する

## イラスト・図解の設計ガイドライン

### 1. 必ずビジュアルを入れるシーン

| シーン | 推奨ビジュアル |
|-------|---------------|
| 課題・問題提起 | 困っている人物イラスト、NGマーク |
| 解決策の提示 | ツールのロゴ/画面、電球アイコン |
| メリット説明 | チェックマーク、上昇グラフ、時計（時短） |
| 比較（Before/After） | 左右対比イラスト、矢印 |
| 手順説明 | 番号付きステップ図、画面キャプチャ |
| 注意事項 | 警告マーク、情報アイコン |
| まとめ | アイコン付きリスト、チェックリスト |

### 2. 図解の種類と使い分け

| 図解タイプ | 用途 | 例 |
|-----------|------|-----|
| フローチャート | プロセス、手順の説明 | 入力→処理→出力 |
| 比較表 | Before/After、選択肢の比較 | 従来 vs AI活用 |
| 放射図 | 中心概念と関連要素 | ツールを中心に活用シーンを配置 |
| タイムライン | 時系列の変化 | 導入前→導入後の変化 |
| ピラミッド/階層図 | 優先度、構造の説明 | 重要度の高い順に配置 |
| マトリクス | 2軸での分類・比較 | 難易度×効果 |

### 3. アイコン・イラストの調達先（参考）

| 種類 | 調達先例 |
|-----|---------|
| アイコン | Flaticon、Icons8、PowerPoint内蔵アイコン、Lucide |
| 人物イラスト | いらすとや、ソコスト、Storyset、unDraw |
| 画面キャプチャ | 実際のツール画面をスクリーンショット |
| フロー図・図解 | PowerPoint/Figmaの図形で作成 |

## 作成手順

### Step 1: 台本の構造分析
1. 台本を読み、セクション（章立て）を把握する
2. 各セクションの**目的**（課題提起/解決策/メリット/手順など）を特定する
3. キーメッセージを抽出する

### Step 2: スライド分割
1. 1セクション = 1スライドを基本とする
2. 情報量が多いセクションは複数スライドに分割する
3. タイトル・まとめ・エンディングを追加する
4. 全体のスライド枚数を確認（目安: 1分あたり1〜2枚）

### Step 3: 各スライドの詳細設計
1. スライドタイプを選択（上記テンプレートから）
2. テキスト内容を簡潔化して記入
3. イラスト・図解の内容を具体的に記述
4. レイアウトを指定

### Step 4: 全体レビュー
1. 流れに一貫性があるか確認
2. ビジュアルの抜け漏れがないか確認
3. 情報量のバランスを確認（詰め込みすぎていないか）

```markdown
### スライド1: [タイトル]

---

---

---

## イラスト・図解の調達方法
[調達先と方法をまとめる]
```

## チェックリスト

スライドアウトライン完成時に確認：

- [ ] すべてのスライドにタイトルがある
- [ ] すべてのスライドにビジュアル要素（イラスト/図解/アイコン）が指定されている
- [ ] 1スライドの情報量が適切（5〜7行以内）
- [ ] 重要キーワードに強調指定がある
- [ ] Before/Afterや比較が必要な箇所に図解がある
- [ ] 手順説明にはステップ番号がある
- [ ] まとめスライドで要点が3つ程度に集約されている
- [ ] 行動を促すCTA（Call To Action）がある
