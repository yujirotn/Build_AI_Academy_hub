## 概要

このガイドは、E-learning講座や研修の台本から、スライド資料の詳細アウトラインを作成するための指示書です。

**v2の主な変更点**: スライド仕様を **COPY / VISUAL / LAYOUT** の3ブロックに分離し、「指示文がスライド本文に混入する事故」を防止します。

---

## 3ブロック分離の基本ルール

### ブロック定義

| ブロック | 役割 | 含めるもの | 表示 |
|---------|------|-----------|------|
| **COPY** | スライドに**そのまま表示する文字** | 見出し、本文、強調テキスト、ラベル | **する** |
| **VISUAL** | **描画する要素**の指示 | アイコン名、イラスト指示、図解構成、色指定 | **しない**（絵として描画） |
| **LAYOUT** | **配置・構成**の指示 | レイアウト種別、分割、余白、配置位置 | **しない**（制作者向け） |

### 重要ルール

1. **COPYには表示テキストのみ**: 「図解：」「アイコン：」「配置：」などの指示語を含めない
2. **VISUALには描画指示のみ**: 「〜です」「〜できます」などの文章を含めない
3. **LAYOUTには配置指示のみ**: 表示するテキストや描画内容を含めない

### NGワードリスト

| ブロック | 含めてはいけない語（混入警告対象） |
|---------|----------------------------------|
| COPY | 「レイアウト」「図解」「アイコン」「配置」「〜形式」「左/右に配置」「イラスト」 |
| VISUAL | 「〜です」「〜ます」「〜できます」「〜しましょう」（文章表現） |
| LAYOUT | 本文テキスト、アイコン名、色指定 |

## 入力と出力

### 入力
- 講座・研修の台本（テキスト形式）

### 出力
- スライドごとの詳細設計書（COPY/VISUAL/LAYOUT形式）

## 台本タイプ別スライド枚数目安

| 台本タイプ | 想定時間 | スライド枚数目安 |
|-----------|---------|----------------|
| ツール解説系 | 15〜20分 | **25〜35枚** |
| ユースケース系 | 10〜15分 | **15〜25枚** |
| 概念系 | 10〜15分 | **15〜20枚** |
| 比較系 | 15〜25分 | **25〜30枚** |

**基本計算**: 1分あたり1.5〜2枚を目安とする

## 詳細化の基本方針

### 1. 導入部分はコンパクトにまとめる

導入部分は以下の4枚構成を基本とする：

| スライド | 内容 |
|---------|------|
| ① こんな経験ありませんか？ | 課題を3つ程度列挙 |
| ② この動画でわかること | 課題がどう改善するか箇条書き（簡潔に） |
| ③ Before/After | 具体的な変化を1枚で凝縮して提示 |
| ④ ツール基本情報 | 料金・対応形式など基本情報をカード形式で |

**※「〇〇とは？」「その悩み、〇〇で解決できます」などの説明スライドは省略可能**

### 2. 重複するスライドは極力なくす

- 同じことを言っているスライドは統合または削除する
- 「概要→個別詳細」のパターンは必須ではない
- 情報量を増やして枚数を減らすことを優先

### 3. 機能紹介は個別スライドでOK

複数の機能/メリットがある場合：

- **個別スライド**: 各項目を1枚ずつ（説明は基本的に不要、要点のみ）
- **概要スライドは省略可能**: 個別スライドだけで十分伝わる場合は不要

### 4. 活用例はシーン別に分割する

活用例は1枚にまとめず、シーン別に分割する。

### 5. 実演部分は簡素でOK

【実演】スライドは、実際の画面操作で補完されるため、簡潔に記載してよい。

### 6. 職種別おすすめを含める

住宅業界向けコンテンツでは、職種別おすすめスライドを追加する。

## 作成手順

### Step 1: 台本の構造分析
1. 台本を読み、セクション（章立て）を把握する
2. 各セクションの**目的**（課題提起/解決策/メリット/手順など）を特定する
3. キーメッセージを抽出する

### Step 2: スライド分割
1. **導入部分はコンパクトに**（課題→わかること→Before/After→基本情報の4枚構成）
2. **重複するスライドは削除**する（同じことを言っているスライドは統合）
3. **機能紹介は個別スライド**に分割（概要スライドは省略可能）
4. **活用例はシーン別**に分割する
5. **実演部分は簡素化**してよい
6. タイトル・まとめ・エンディングを追加する
7. 全体のスライド枚数を確認

### Step 3: 各スライドの3ブロック設計
1. スライドタイプを選択（上記テンプレートから）
2. **COPY**: 表示するテキストのみを記入
3. **VISUAL**: 描画要素の指示を記入
4. **LAYOUT**: 配置指示を記入
5. **notes**: 制作者向けメモを記入

### Step 4: 混入チェック
1. COPYにNGワード（レイアウト、図解、アイコン等）が含まれていないか確認
2. VISUALに文章表現が含まれていないか確認
3. LAYOUTに表示テキストが含まれていないか確認

### Step 5: 全体レビュー
1. 流れに一貫性があるか確認
2. ビジュアルの抜け漏れがないか確認
3. 情報量のバランスを確認
4. 職種別おすすめが含まれているか確認

## 補足：従来形式からの移行方法

従来のMarkdown表形式からYAML形式への移行は、以下のタグ付けで段階的に行えます：

```markdown
### スライドN: [タイトル]

[LAYOUT] 中央配置
[LAYOUT] 3カラム

[COPY] 見出し: まとめ
[COPY] カード①: 情報検索が瞬時に完了
[COPY] カード②: 概算計算が自動化

[VISUAL] カード①アイコン: search
[VISUAL] カード②アイコン: calculator

[NOTES] 色はネイビー基調
```

この形式であれば、従来の書き方を維持しつつ、3ブロック分離の恩恵を受けられます。
