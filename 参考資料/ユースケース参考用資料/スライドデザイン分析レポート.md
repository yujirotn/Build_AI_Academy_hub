### セクション区切りスライド

各セクション間に挿入される区切りスライドのパターン：

- **概要** - ツール説明セクションの開始
- **メリット** - メリット解説セクションの開始
- **実演** - 実演デモセクションの開始

## 3. 各スライドタイプの詳細解説

### 3.1 タイトルスライド

**構成要素：**
- ロゴ（Build AI Academy）：左上
- メインタイトル：中央大きく配置
- サブタイトル/キャッチコピー：タイトル下
- バッジ類：「初心者向け」「2025年12月版」など
- 背景：薄いグラデーションまたはパターン

**デザイン特徴：**
- タイトルは2行以内に収める
- ツール名は太字または別カラーで強調
- バッジは右上または左下に配置

### 3.2 課題提示スライド（こんな経験ありませんか？）

**構成要素：**
- 見出し：「こんな経験ありませんか？」
- 課題リスト：3〜4つの具体的な悩み
- イラスト：困っている人物のイラスト

**表現パターン：**
```
・〇〇で△△に時間がかかる...
・□□の確認が大変...
・××を探すのが面倒...
```

### 3.3 解決策提示スライド

**構成要素：**
- 見出し：「その悩み、〇〇で解決できます」
- ツールロゴまたはアイコン
- 簡潔な解決策の説明（1〜2文）

**デザイン特徴：**
- 明るい背景色（オレンジ系アクセント）
- ポジティブなイメージのイラスト

### 3.4 この動画でわかること

**構成要素：**
- 見出し：「この動画でわかること」
- 学習内容リスト：3〜4項目
- チェックマークアイコン

**レイアウト：**
- 左側にアイコン/イラスト
- 右側にリスト形式で内容

### 3.5 ツール概要スライド（〇〇とは？）

**構成要素：**
- 見出し：「〇〇とは？」
- ツールの説明文（2〜3文）
- スクリーンショットまたはUIイメージ
- 提供元情報（Google、OpenAIなど）

### 3.6 基本情報/制限事項スライド

**構成要素：**
- 料金プラン情報
- 利用制限（文字数、ファイルサイズなど）
- 対応フォーマット
- 注意事項

**レイアウト：**
- 表形式または箇条書き
- アイコン付きでカテゴリ分け

### 3.7 メリットまとめスライド

**構成要素：**
- 見出し：「活用メリット」
- 3つのメリットをカード形式で配置
- 各カードにアイコンと短いキャッチコピー

**レイアウト：**
- 横3列のカード配置
- 各カードは同サイズ・同形式

### 3.8 メリット詳細スライド（①②③）

**構成要素：**
- 番号付き見出し：「メリット①：〇〇」
- Before/After比較
- 具体的な効果の説明
- ビジュアル（イラスト、図解）

**Before/After形式：**
```
┌──────────────────┐    ┌──────────────────┐
│    Before        │ → │     After        │
│   従来の方法      │    │   ツール活用後    │
│   課題・問題点    │    │   解決された状態  │
└──────────────────┘    └──────────────────┘
```

### 3.9 全体の流れスライド

**構成要素：**
- 見出し：「全体の流れ」
- フロー図（横並び3〜5ステップ）
- 各ステップの簡潔な説明

**フロー図パターン：**
```
[Step1] → [Step2] → [Step3] → [Step4] → [完成]
 資料      質問      回答      活用
アップロード  入力     確認     する
```

### 3.10 実演スライド

**構成要素：**
- 見出し：「【実演】ステップ○：〇〇」
- スクリーンショット（大きめ）
- 操作手順の説明
- ポイント・注意点

**デザイン特徴：**
- スクリーンショットが主役（画面の60%以上）
- 操作箇所を赤枠や矢印でハイライト
- 説明テキストは最小限

### 3.11 職種別おすすめスライド

**構成要素：**
- 見出し：「職種別おすすめの使い方」
- 3つの職種カード（設計/営業/現場など）
- 各職種の活用シーン

**レイアウト：**
- 横3列のカード配置
- 職種アイコン + 活用例

## 4. デザインシステム

### 4.1 カラーパレット

| 用途 | カラー | カラーコード（推定） |
|------|--------|-------------------|
| メインカラー | ネイビー | #1E3A5F |
| アクセントカラー | オレンジ | #FF6B35 |
| 背景色（明） | ホワイト/ライトグレー | #FFFFFF / #F5F5F5 |
| 背景色（暗） | ダークグレー | #333333 |
| テキスト | ダークグレー | #333333 |
| 強調テキスト | オレンジ | #FF6B35 |

### 4.2 使用カラーのルール

- **ネイビー**：見出し、タイトル、重要な枠線
- **オレンジ**：アクセント、CTA、強調したいポイント
- **ホワイト/グレー**：背景、カード背景
- **ダークグレー**：本文テキスト

### 4.3 フォント

- **見出し**：太めのゴシック体（推定：Noto Sans JP Bold）
- **本文**：標準ゴシック体（推定：Noto Sans JP Regular）
- **強調**：太字 + オレンジカラー

### 4.4 アイコン・イラスト

**使用パターン：**
- フラットデザインのアイコン
- シンプルな人物イラスト
- ツールのロゴ・UIスクリーンショット

**アイコンの役割：**
- リスト項目の視認性向上
- セクション区切りの明示
- 職種・機能の象徴

## 5. ビジュアル表現パターン

### 5.1 Before/After比較

**用途：** メリット説明、効果の可視化

```
┌─────────────────────────────────────────────────────┐
│  Before（従来）         →        After（改善後）     │
│  ┌──────────────┐            ┌──────────────┐      │
│  │ 時間がかかる   │            │ 時短できる    │      │
│  │ 手間が多い    │     →      │ 自動化      │      │
│  │ ミスが起きる   │            │ 正確        │      │
│  └──────────────┘            └──────────────┘      │
└─────────────────────────────────────────────────────┘
```

### 5.2 フロー図（横並び）

**用途：** 全体の流れ、手順説明

```
[1]────→[2]────→[3]────→[4]
資料      質問     回答     完成
準備      入力     確認
```

### 5.3 カード形式（3列）

**用途：** メリットまとめ、職種別説明

```
┌────────┐  ┌────────┐  ┌────────┐
│ 🏠設計  │  │ 📈営業  │  │ 🏗️現場  │
│        │  │        │  │        │
│ 活用例  │  │ 活用例  │  │ 活用例  │
└────────┘  └────────┘  └────────┘
```

### 5.4 リスト形式（チェックマーク付き）

**用途：** 学習内容、課題リスト

```
✓ 〇〇の基本的な使い方
✓ △△を効率化する方法
✓ □□の実践テクニック
```

### 5.5 スクリーンショット + 説明

**用途：** 実演スライド

```
┌─────────────────────────────────────┐
│                                     │
│    [スクリーンショット（大）]         │
│         ↑ 赤枠でハイライト           │
│                                     │
├─────────────────────────────────────┤
│ ポイント：〇〇をクリック              │
└─────────────────────────────────────┘
```

## 6. キーメッセージの配置ルール

### 6.1 見出しの配置

- **位置**：スライド上部（上から1/4の位置）
- **サイズ**：本文の1.5〜2倍
- **装飾**：下線、背景色、アイコン付き

### 6.2 強調ポイントの配置

- **キャッチコピー**：中央または左寄せ大きく
- **数字・効果**：オレンジ色で強調
- **補足説明**：本文サイズで下部に配置

### 6.3 CTA（行動喚起）の配置

- **位置**：スライド下部
- **形式**：ボタン風デザインまたは矢印付き
- **文言例**：「詳しくはこちら」「今すぐ試す」

## 7. スライド枚数と構成比率

### 7.1 標準的な枚数配分

| セクション | 枚数 | 比率 |
|-----------|------|------|
| 導入（タイトル〜この動画でわかること） | 4枚 | 20% |
| 概要（ツール説明〜基本情報） | 2枚 | 10% |
| メリット | 4枚 | 20% |
| 実演 | 5〜8枚 | 30〜40% |
| まとめ（職種別〜エンディング） | 3枚 | 15% |

### 7.2 全体枚数の目安

- **最小構成**：12枚
- **標準構成**：15〜16枚
- **詳細構成**：18〜20枚

## 8. 今後の制作への推奨事項

### 8.1 テンプレート化すべき要素

1. **タイトルスライド**のレイアウト（ロゴ位置、バッジ配置）
2. **Before/After比較**のフォーマット
3. **メリットカード**の3列レイアウト
4. **フロー図**のデザインパターン
5. **職種別カード**のフォーマット

### 8.2 統一すべきルール

- カラーパレットの厳守（ネイビー + オレンジ）
- 見出しフォントサイズの統一
- スクリーンショットのハイライト方法
- アイコンスタイルの統一

### 8.3 改善検討ポイント

- セクション区切りスライドのデザイン強化
- 実演スライドの操作説明の見やすさ向上
- モバイル視聴を意識した文字サイズ調整
